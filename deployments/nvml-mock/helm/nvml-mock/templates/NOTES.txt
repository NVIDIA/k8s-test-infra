OPTION B: NVIDIA DRA Driver (requires DRA-enabled cluster)

  Cluster prerequisites (KIND):
    kind create cluster --config tests/e2e/kind-dra-config.yaml

  Install:
    helm repo add nvidia https://helm.ngc.nvidia.com/nvidia
    helm repo update

    helm install nvidia-dra-driver nvidia/nvidia-dra-driver-gpu \
      --namespace nvidia \
      --create-namespace \
      --set nvidiaDriverRoot=/var/lib/nvml-mock/driver \
      --set gpuResourcesEnabledOverride=true \
      --set resources.computeDomains.enabled=false \
      --wait --timeout 180s

  Verify ResourceSlices:

    kubectl get resourceslices -o json | \
      jq '[.items[].spec.devices // [] | length] | add // 0'

OPTION C: GPU Operator (tested in CI -- three modes)

  Driver disabled (default -- this chart's /run/nvidia/driver symlink serves
  the operator validator):

    helm install gpu-operator nvidia/gpu-operator \
      -n gpu-operator --create-namespace \
      -f tests/e2e/gpu-operator-values.yaml

  Operator-managed mock driver: install this chart with
  --set gpuOperator.driverSymlink.enabled=false, then follow
  docs/mock-driver.md (mock-driver image tagged for your nodes' OS,
  layered values files, operator version pin).

  Host driver masquerade: install this chart with --set hostDriver.enabled=true
  (nvidia-smi + mock libs at standard host paths; zero operator env overrides),
  then layer tests/e2e/go/assets/gpu-operator-hostdriver-values.yaml. Test
  nodes only. See docs/mock-driver.md.
