# Node-Wide Mock GPU Injection Design

**Date:** 2026-06-30  
**Status:** Approved  
**Authors:** Design session (brainstorming)

## Summary

Extend nvml-mock so that workloads beyond the DaemonSet pod can see a simulated
GPU node. Today the DaemonSet writes mock artifacts to the host filesystem and
only processes inside that pod (or CDI-injected GPU workloads) see NVML and IB
mocks. This design adds tiered CDI specs and a mutating admission webhook so
GPU Operator components, GPU-requesting pods, and opt-in discovery pods all
receive the appropriate mock environment on Kind clusters.

DCGM hostengine integration is explicitly deferred.

## Goals

- GPU Operator stack (validator, device plugin, GFD) continues to work unchanged.
- Pods requesting `nvidia.com/gpu` automatically receive the full mock stack
  (NVML, IB, PCI sysfs redirect, device nodes).
- Other pods can opt in via annotation with configurable depth:
  `nvml-mock.nvidia.com/tier: nvml|ib|full`.
- Discovery tools (`nvidia-smi`, `ibstat`, `iblinkinfo`, `ibnetdiscover`) work
  inside opt-in pods without `kubectl exec` into the DaemonSet.
- Single source of truth for injection: CDI specs generated by `setup.sh`.

## Non-Goals

- DCGM hostengine / dcgm-exporter enablement (follow-up).
- CUDA kernel execution fidelity (mock `libcuda.so` remains early-stage).
- Automatic injection into every pod on the node.
- MIG or SR-IOV VF simulation.
- Production cluster hardening beyond portable CDI + webhook patterns.

## Background

### Current state

The nvml-mock DaemonSet (`deployments/nvml-mock/scripts/setup.sh`) writes to
the host:

| Path | Purpose |
|------|---------|
| `/var/lib/nvml-mock/driver/` | Mock NVML/CUDA libs, `nvidia-smi`, `/dev/nvidia*` nodes |
| `/var/lib/nvml-mock/ib/` | Fake InfiniBand sysfs tree |
| `/var/lib/nvml-mock/sys/` | Fake PCI sysfs tree (`render-pci-sysfs`) |
| `/var/run/cdi/nvidia.yaml` | CDI spec for GPU device injection |
| `/run/nvidia/driver` | GPU Operator compatibility symlink |

Inside the DaemonSet pod only, `LD_PRELOAD` activates IB shims
(`libibmocksys`, `libibmockumad`, `libibmockverbs`). CDI-injected GPU pods get
NVML libs and device nodes but not IB shims or PCI redirect. Arbitrary pods see
nothing.

### Requirements (from design session)

| Decision | Choice |
|----------|--------|
| Success bar | Full stack minus DCGM: GPU Operator + arbitrary user pods |
| Injection model | Tiered: auto for GPU Operator + `nvidia.com/gpu`; opt-in for others |
| Opt-in mechanism | Annotation `nvml-mock.nvidia.com/tier: nvml\|ib\|full` |
| DCGM | Deferred |

## Architecture

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Kind Node (host filesystem)                                              │
│  nvml-mock DaemonSet                                                     │
│    → /var/lib/nvml-mock/{driver,ib,sys,run/}                            │
│    → /var/run/cdi/{nvidia,nvml-mock-*.yaml}                             │
└─────────────────────────────────────────────────────────────────────────┘
         ▲                                    │
         │ hostPath / CDI bind-mounts          │ CDI resolver (containerd)
         │                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Kubernetes                                                               │
│  nvml-mock-injector (mutating webhook)                                   │
│    → maps tier rules → cdi.k8s.io/* annotations                         │
│    → sets runtimeClassName: nvidia when needed                          │
│                                                                          │
│  Workloads: GPU Operator | nvidia.com/gpu pods | opt-in discovery pods  │
└─────────────────────────────────────────────────────────────────────────┘
```

**Principle:** The DaemonSet owns host-side artifacts. The webhook decides
which CDI devices a pod receives. CDI performs the actual container edits.

## Tier Matrix

| Tier | Trigger | nvidia-smi | IB tools | PCI sysfs | /dev/nvidia* |
|------|---------|:----------:|:--------:|:---------:|:------------:|
| `gpu` | Auto: GPU Operator pods + `nvidia.com/gpu` requests | ✓ | ✓ | ✓ | ✓ |
| `full` | `tier=full` annotation | ✓ | ✓ | ✓ | ✗ |
| `ib` | `tier=ib` annotation | ✓ | ✓ | ✗ | ✗ |
| `nvml` | `tier=nvml` annotation | ✓ | ✗ | ✗ | ✗ |

Tier composition is cumulative: `ib` includes `nvml`; `full` includes `ib`;
`gpu` includes `full` plus per-GPU device nodes.

## Components

### 1. Tiered CDI spec generation (`setup.sh`)

`setup.sh` writes CDI specs to `/var/run/cdi/` on the host:

| File | CDI kind | Devices |
|------|----------|---------|
| `nvidia.yaml` | `nvidia.com/gpu` | `0`…`N`, `all` — extended with IB + PCI + fabricmanager |
| `nvml-mock-nvml.yaml` | `nvml-mock.nvidia.com/mock` | `nvml` |
| `nvml-mock-ib.yaml` | `nvml-mock.nvidia.com/mock` | `ib` |
| `nvml-mock-full.yaml` | `nvml-mock.nvidia.com/mock` | `full` |

#### `nvml` device — containerEdits

- Bind-mount mock `libnvidia-ml.so*`, optional `libcuda.so*`, `nvidia-smi`.
- Bind-mount `/var/lib/nvml-mock/driver/config/config.yaml` →
  `/etc/nvml-mock/config.yaml`.
- Bind-mount `/proc/driver/nvidia/version` and `params` stubs.
- Env: `MOCK_NVML_CONFIG=/etc/nvml-mock/config.yaml`.
- Hook: `nvidia-cdi-hook update-ldcache --folder /usr/lib64`.

#### `ib` device — adds to `nvml`

- Bind-mount IB shim libraries to `/usr/local/lib/`:
  `libibmockumad.so.1`, `libibmockverbs.so.1`, `libibmocksys.so.1`.
- Bind-mount `/var/lib/nvml-mock/ib` → `/var/lib/nvml-mock/ib`.
- Bind-mount `/var/lib/nvml-mock/run/mock-ib.sock` (host socket).
- Env:
  - `LD_PRELOAD=/usr/local/lib/libibmockumad.so.1:/usr/local/lib/libibmockverbs.so.1:/usr/local/lib/libibmocksys.so.1`
  - `MOCK_IB_ROOT=/var/lib/nvml-mock/ib`
  - `MOCK_IB=full`
  - `MOCK_IB_PING_SOCKET=/var/lib/nvml-mock/run/mock-ib.sock`

#### `full` device — adds to `ib`

- Bind-mount `libpcimocksys.so.1` → `/usr/local/lib/`.
- Env: `MOCK_PCI_ROOT=/var/lib/nvml-mock`.
- Append `libpcimocksys.so.1` to `LD_PRELOAD` chain.

#### `gpu` tier — extends existing `nvidia.yaml`

All `full`-tier edits plus existing per-GPU device node entries and
fabricmanager state directory when enabled.

### 2. PCI sysfs redirect shim (new)

**Location:** `pkg/system/mockpcisysfs/c/shim.c` → `libpcimocksys.so.1`

Mirrors `pkg/network/mockib/c/shim.c`:

- Hooks libc path functions: `open`, `openat`, `readlink`, `readlinkat`,
  `stat`, `lstat`, `fstatat`, `access`, `faccessat`, etc.
- Rewrites paths under `/sys/bus/pci/` and `/sys/devices/pci*` to
  `$MOCK_PCI_ROOT/sys/...` via `dlsym(RTLD_NEXT, ...)`.
- No-op when `MOCK_PCI_ROOT` is unset.

The rendered tree at `/var/lib/nvml-mock/sys/` already exists via
`render-pci-sysfs`; the shim exposes it inside containers without bind-mounting
over `/sys`.

### 3. Host-exposed mock-ib socket

**Change:** Default `MOCK_IB_PING_SOCKET` moves from `/run/mock-ib.sock`
(pod-local) to `/var/lib/nvml-mock/run/mock-ib.sock` (hostPath-backed).

- `start-mock-ib.sh` listens on the host path.
- DaemonSet ensures `/var/lib/nvml-mock/run/` exists on the host.
- CDI `ib`/`full`/`gpu` tiers bind-mount the socket into consumer pods at the
  same path so UMAD shims reach the DaemonSet's `mock-ib` daemon.

Cross-pod IB tools (`ibnetdiscover`, `ibping`) require the shared daemon plus
TCP fabric relay (existing `mock-ib -fabric` behavior).

### 4. Mutating admission webhook (`cmd/nvml-mock-injector`)

Deployed via Helm: Deployment, Service, TLS (cert-manager or chart-generated
certs), MutatingWebhookConfiguration, RBAC.

#### Scope

Only mutate pods scheduled on nodes labeled `nvidia.com/gpu.present=true`.

#### Tier resolution (first match wins)

1. **Auto `gpu`:** Pod carries a GPU Operator operand label
   (`app.kubernetes.io/component` in a configured allowlist: `nvidia-device-plugin`,
   `nvidia-gpu-feature-discovery`, `nvidia-operator-validator`, etc.).
2. **Auto `gpu`:** Pod spec requests `nvidia.com/gpu` resource (limits or
   requests).
3. **Opt-in:** Annotation `nvml-mock.nvidia.com/tier` set to `nvml`, `ib`, or
   `full`.
4. **No-op:** All other pods.

Validation rejects:

- Unknown tier values.
- `tier=gpu` via annotation (GPU tier is auto-only).

#### Mutation

Add native Kubernetes CDI device annotations (containerd `enable_cdi = true`):

```yaml
# tier=full (opt-in)
cdi.k8s.io/nvml-mock.nvidia.com-mock: "full"

# tier=gpu (auto)
cdi.k8s.io/nvml-mock.nvidia.com-mock: "full"
cdi.k8s.io/nvidia.com-gpu: "all"   # or specific GPU index from resource request
```

Set `spec.runtimeClassName: nvidia` when not already set and tier requires CDI
injection via nvidia-container-runtime (Kind GPU Operator E2E config).

Record an audit annotation `nvml-mock.nvidia.com/injected-tier: <tier>` on
mutated pods for debugging.

### 5. Helm chart additions

```yaml
injector:
  enabled: true
  replicaCount: 1
  # TLS, resources, tolerations

  gpuOperator:
    # Labels/components that trigger auto gpu tier
    operandComponents:
      - nvidia-device-plugin
      - gpu-feature-discovery
      - nvidia-operator-validator
```

Webhook can share the nvml-mock container image (binary at
`/usr/local/bin/nvml-mock-injector`) or ship as a separate slim image.

## Data Flow

### Opt-in discovery pod

```
1. User creates pod with nvml-mock.nvidia.com/tier: ib
2. API server calls mutating webhook
3. Webhook verifies target node has nvidia.com/gpu.present=true
4. Webhook adds cdi.k8s.io/nvml-mock.nvidia.com-mock: "ib"
5. Webhook sets runtimeClassName: nvidia
6. Scheduler places pod; kubelet creates container
7. containerd CDI resolver applies nvml-mock-ib.yaml edits
8. Container starts with mock NVML + IB shims
9. nvidia-smi -L and ibstat succeed
```

### GPU workload pod

```
1. User creates pod with resources.limits.nvidia.com/gpu: 1
2. Webhook adds gpu tier CDI annotations + nvidia.com/gpu device
3. Device plugin (mock NVML) has advertised nvidia.com/gpu on the node
4. Pod receives /dev/nvidia0 + full mock stack
5. nvidia-smi shows allocated GPU
```

### DaemonSet startup (extended)

```
setup.sh:
  render IB sysfs, PCI sysfs, NVML/CUDA libs, device nodes
  start mock-ib on /var/lib/nvml-mock/run/mock-ib.sock
  write tiered CDI specs to /var/run/cdi/
  label node nvidia.com/gpu.present=true
  (existing: /run/nvidia/driver symlink, toolkit-ready marker)
```

## Error Handling

| Condition | Behavior |
|-----------|----------|
| Webhook unavailable | Fail closed for pods with tier annotation; unannotated pods unaffected |
| Invalid tier annotation | Webhook rejects with clear HTTP 400 message |
| CDI spec missing on node | Container create fails; DaemonSet liveness should verify CDI files |
| mock-ib socket not ready | IB tools fail at runtime; callers retry (existing E2E pattern) |
| Tier annotation on non-mock node | Webhook skips mutation; pod runs without mock (user misconfiguration) |
| `MOCK_IB=off` in pod env | Shims no-op (existing escape hatch) |

## Testing Plan

| Test | Validates |
|------|-----------|
| E2E: ubuntu pod + `tier=nvml` | `nvidia-smi -L` works; no `/dev/nvidia0` |
| E2E: ubuntu pod + `tier=ib` | `ibstat` lists mock HCAs |
| E2E: ubuntu pod + `tier=full` | PCI readlink resolves inside pod |
| E2E: pod with `nvidia.com/gpu: 1` | Auto-injection; device node present |
| E2E: GPU Operator validator | Regression — still passes |
| E2E: two opt-in `tier=ib` pods | Cross-pod `ibnetdiscover` visibility |
| Helm unittest | CDI template output, webhook mutation rules |
| Unit: `libpcimocksys.so` | Path rewrite for `/sys/bus/pci/devices/*` |

## Implementation Order

1. Move mock-ib socket to host path; update DaemonSet and E2E scripts.
2. Extend `nvidia.yaml` CDI generation with IB mounts/env.
3. Implement `libpcimocksys.so` + unit tests.
4. Add tiered CDI spec generation in `setup.sh`.
5. Build `cmd/nvml-mock-injector` webhook + Helm wiring.
6. Add per-tier E2E tests.
7. Update Helm README and quickstart docs.
